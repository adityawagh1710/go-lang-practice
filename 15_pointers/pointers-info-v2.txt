✅ What is a Pointer?
   • A pointer holds the MEMORY ADDRESS of a value.
   • Type: *T (pointer to type T)
   • Zero value: nil

✅ Why Use Pointers?
   • Avoid copying large structs/slices/maps
   • Mutate original data (pass by reference)
   • Share data across functions/goroutines

✅ Basic Syntax
   var p *int          // pointer declaration (nil)
   x := 42
   p = &x              // & = address-of operator
   fmt.Println(*p)     // * = dereference operator → prints 42

   *p = 100            // changes original x to 100

💡 Medior Tip: Always check for nil before dereferencing!
   if p != nil { ... }

📌 Example 1: Swap Function (Classic Interview Starter)
```go
package main
import "fmt"

func swap(a, b *int) {
    *a, *b = *b, *a
}

func main() {
    x, y := 10, 20
    fmt.Printf("Before: x=%d, y=%d\n", x, y)
    swap(&x, &y)
    fmt.Printf("After:  x=%d, y=%d\n", x, y)
}

📌 Example 2: Pointer to Struct (Efficient Mutation)

type User struct {
    Name string
    Age  int
}

func updateAge(u *User, newAge int) {
    if u != nil {
        u.Age = newAge
    }
}

func main() {
    user := &User{Name: "Alice", Age: 25}
    updateAge(user, 30)
    fmt.Printf("%+v\n", user) // &{Name:Alice Age:30}
}

📌 Example 3: Returning Pointers from Functions (Factory Pattern)

func NewUser(name string, age int) *User {
    return &User{Name: name, Age: age} // safe – Go escapes to heap
}

🔥 Advanced Concepts
1️⃣ Pointer Receivers vs Value Receivers
• Value receiver → copy (safe for small types)
• Pointer receiver → mutation + performance (use for large structs)
Gotype Counter struct{ value int }

// Value receiver – does NOT mutate
func (c Counter) Increment() { c.value++ }

// Pointer receiver – DOES mutate
func (c *Counter) Increment() { c.value++ }
2️⃣ Escape Analysis
• Go compiler decides: stack vs heap
• Returning &localVar is OK – compiler escapes to heap automatically
3️⃣ Nil Pointers & Interface Gotchas
Govar p *int          // p == nil
var i interface{} = p // i != nil (contains typed nil!)
4️⃣ unsafe.Pointer (Senior Only – Use with Care!)
Goimport "unsafe"

func bytesToString(b []byte) string {
    return *(*string)(unsafe.Pointer(&b)) // zero-copy hack
}
5️⃣ Pointers in Goroutines (Thread Safety)
• Never share pointers without sync.Mutex or channels
• Use atomic package for simple cases
📌 Senior Example: Concurrent Counter with Pointer
Goimport (
    "sync"
    "sync/atomic"
)

type AtomicCounter struct{ value int64 }

func (c *AtomicCounter) Inc() {
    atomic.AddInt64(&c.value, 1)
}

║  SECTION 4: COMMON PITFALLS & BEST PRACTICES                               ║ 
╚════════════════════════════════════════════════════════════════════════════╝
❌ Pitfall #1: Returning pointer to stack variable → DANGER in other langs
✅ In Go: Compiler handles escape analysis → safe
❌ Pitfall #2: Forgetting nil check
✅ Always: if ptr != nil
❌ Pitfall #3: Using pointers for small types (int, bool)
✅ Bad for performance & readability
✅ Best Practices:
• Use pointer receivers for structs > 8 bytes
• Prefer value semantics unless mutation is required
• Document when you return pointers
• Use go vet & golangci-lint to catch pointer issues
