# 🚀 Go Language: Defer, Panic, Recover — Complete Notes

---

# 📌 1. DEFER

## 🔹 What is `defer`?

* `defer` schedules a function call to run **after the surrounding function completes**
* Executed in **LIFO (Last In First Out)** order
* Commonly used for **cleanup tasks**

---

## 🔹 Basic Example

```go
package main

import "fmt"

func main() {
    defer fmt.Println("World")
    fmt.Println("Hello")
}
```

### ✅ Output:

```
Hello
World
```

---

## 🔹 Multiple Defers (LIFO Order)

```go
func main() {
    defer fmt.Println("1")
    defer fmt.Println("2")
    defer fmt.Println("3")
}
```

### ✅ Output:

```
3
2
1
```

---

## 🔹 Real-world Example (File Handling)

```go
file, err := os.Open("file.txt")
if err != nil {
    panic(err)
}
defer file.Close()
```

✔ Ensures file is always closed
✔ Even if function exits early

---

## 🔹 Defer with Arguments (Important!)

```go
func main() {
    x := 10
    defer fmt.Println(x)
    x = 20
}
```

### ✅ Output:

```
10
```

👉 Arguments are evaluated **at defer time**, not execution time

---

## 🔹 Defer in Loops (Common Mistake)

```go
for i := 0; i < 3; i++ {
    defer fmt.Println(i)
}
```

### ❗ Output:

```
2
1
0
```

---

## 🔹 Best Practices

✔ Use for cleanup (files, DB, locks)
✔ Avoid heavy logic inside defer
✔ Be careful in loops (can cause memory issues)

# 📌 2. PANIC

## 🔹 What is `panic`?

* Stops normal execution of the program
* Begins **stack unwinding**
* Executes all deferred functions before crashing

---

## 🔹 Basic Example

```go
func main() {
    panic("Something went wrong!")
}
```

### ❌ Output:

```
panic: Something went wrong!
```

---

## 🔹 Panic with Defer

```go
func main() {
    defer fmt.Println("Cleanup done")
    panic("Crash!")
}
```

### ✅ Output:

```
Cleanup done
panic: Crash!
```

---

## 🔹 When to Use Panic?

✔ Irrecoverable errors
✔ Programmer mistakes
✔ Invalid state

❌ Not for normal error handling (use `error` instead)

---

## 🔹 Custom Panic Example

```go
func divide(a, b int) int {
    if b == 0 {
        panic("division by zero")
    }
    return a / b
}
```

---

# 📌 3. RECOVER

## 🔹 What is `recover`?

* Used to **catch panic and prevent crash**
* Must be called inside a **deferred function**

---

## 🔹 Basic Example

```go
func main() {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("Recovered:", r)
        }
    }()

    panic("Oops!")
}
```

### ✅ Output:

```
Recovered: Oops!
```

---

## 🔹 Important Rule

🚨 `recover()` only works inside `defer`

```go
recover() // ❌ won't work outside defer
```

---

## 🔹 Real-world Example (Safe Execution)

```go
func safeExecute() {
    defer func() {
        if err := recover(); err != nil {
            fmt.Println("Error handled:", err)
        }
    }()

    fmt.Println("Start")
    panic("Failure")
    fmt.Println("End") // won't execute
}
```

---

# 📌 4. Execution Flow (IMPORTANT)

```
Function starts
   ↓
Defer registered
   ↓
Panic occurs
   ↓
Deferred functions execute
   ↓
Recover (if exists)
   ↓
Program continues OR crashes
```

---

# 📌 5. Combined Example (Defer + Panic + Recover)

```go
func risky() {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("Recovered in risky():", r)
        }
    }()

    fmt.Println("Executing risky function")
    panic("Something bad happened")
}

func main() {
    risky()
    fmt.Println("Program continues...")
}
```

### ✅ Output:

```
Executing risky function
Recovered in risky(): Something bad happened
Program continues...
```

# 📌 6. Interview Questions & Answers

---

## ❓ Q1: What is `defer` used for?

✅ Answer:
To delay execution of a function until the surrounding function returns. Commonly used for cleanup.

---

## ❓ Q2: In what order are deferred calls executed?

✅ Answer:
LIFO (Last In First Out)

---

## ❓ Q3: When are defer arguments evaluated?

✅ Answer:
At the time `defer` is declared, not when executed.

---

## ❓ Q4: What is `panic`?

✅ Answer:
A built-in function that stops normal execution and begins stack unwinding.

---

## ❓ Q5: Does `panic` execute deferred functions?

✅ Answer:
Yes, all deferred functions run before the program crashes.

---

## ❓ Q6: What is `recover`?

✅ Answer:
A function that catches a panic and prevents the program from crashing.

---

## ❓ Q7: Where must `recover` be used?

✅ Answer:
Inside a deferred function only.

---

## ❓ Q8: Can we use panic for normal error handling?

❌ Answer: No
✔ Use `error` instead

---

## ❓ Q9: What happens if recover is not used?

✅ Answer:
Program crashes after executing deferred functions.

---

## ❓ Q10: Real-world use of recover?

✅ Answer:

* Web servers (prevent crashing)
* Middleware error handling
* Safe goroutine execution

---

# 📌 7. Senior-Level Insights 💡

✔ Use `defer` for:

* Mutex unlocks (`defer mu.Unlock()`)
* File closing
* DB connections

✔ Avoid `panic` in:

* Business logic
* API responses

✔ Use `recover` in:

* HTTP middleware
* Worker pools
* Goroutines
